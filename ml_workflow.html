1Data Collection

Gather raw data from the relevant sources. The quality, quantity and relevance of the collected data set the upper limit on how well any model can perform.

Sources

Databases, CSV/Excel files, APIs, web scraping, sensors/IoT, surveys, public datasets.

Goals

Sufficient volume, representative samples, minimal collection bias, correct labels (if supervised).

Checks

Duplicates, leakage, time alignment, units consistency, schema validation.

2EDA — Exploratory Data Analysis

Understand the data before modeling: is it labelled or unlabelled, and is the task classification or regression? Then visualize and study the distributions of numeric columns.

Data Visualization & Distributions

For each numeric column, determine whether it is normally distributed, skewed, or follows some other distribution — this decision drives later choices (imputation, outlier rule, scaling).

GoalPlot
Distribution of a numeric columnHistogram, KDE / density plot
Spread & outliersBox plot, violin plot
Relationship between two numericsScatter plot
Correlation among many featuresHeatmap, pair plot
Category frequencyBar / count plot
Normality checkQ-Q plot
Distribution shapes: A symmetric bell curve → Normal. A long tail on one side → Skewed (right/positive or left/negative). Neither → Other. The shape decides the imputation value and outlier-detection formula used in Feature Engineering.

c) Outlier Detection & Treatment

First detect the valid range based on the distribution, then treat the points that fall outside it.

DistributionValid range (bounds)Rationale
Normally distributed[ mean − 3·STD , mean + 3·STD ]~99% of data lies within ±3σ
Skewed distribution[ Q1 − 1.5·IQR , Q3 + 1.5·IQR ]IQR-based fences (Tukey's method)
Neither skewed nor normal[ 2.5th percentile , 97.5th percentile ]Percentile (winsorizing) bounds

IQR = Q3 − Q1

TreatmentWhat it does
TrimmingOnce the range is known, remove all rows whose values fall outside it.
CappingOnce the range is known, replace values outside it with the boundary value (lower/upper bound).

d) Feature Scaling

TechniqueWhen to applyTypical range
Normalization (Min-Max)When values now and in the future are known to stay within a specified, bounded range[0, 1]
Standardization (Z-score)When the range of the numeric column is not known in advancemean 0, std 1
Normalization: x' = (x − min) / (max − min)
Standardization: x' = (x − mean) / STD

3.2 — Feature Construction

Create new attributes from existing ones using domain knowledge (e.g. BMI from height & weight, "price per sq.ft" from price and area, day-of-week from a date).

3.3 — Feature Selection

Select only the important features for model training — reduces noise, overfitting and training time.

Filter methods (correlation, chi²) Wrapper methods (RFE) Embedded methods (Lasso, tree importance)

3.4 — Feature Extraction

PCA (Principal Component Analysis) — dimensionality reduction that combines multiple features into fewer composite features for analysis, while retaining most of the variance.

Selection keeps a subset of original features; Extraction creates new combined features.

4Data Splitting

Partition the prepared data so the model is evaluated on data it never saw during training.

Training set fit the model
Validation set tune & compare
Test set final unbiased check

Common split ratios: 70/15/15 or 80/10/10. Use stratified splitting for imbalanced classification.

6Model Training & K-Fold Cross Validation

Train the selected models, then use K-Fold Cross Validation to reliably estimate performance and pick the best model — instead of trusting a single train/test split.

How K-Fold Works (K = 5 example)

IterationFold 1Fold 2Fold 3Fold 4Fold 5
1TestTrainTrainTrainTrain
2TrainTestTrainTrainTrain
3TrainTrainTestTrainTrain
4TrainTrainTrainTestTrain
5TrainTrainTrainTrainTest

The final score is the average across all K folds — giving a robust estimate to compare and select the best model.

7Hyperparameter Tuning

Having chosen the model, try it out with different parameter settings to squeeze out the best performance.

MethodIdea
Grid SearchExhaustively try every combination in a defined grid
Random SearchSample random combinations — faster over large spaces
Bayesian OptimizationUse past results to choose promising next settings

Examples of hyperparameters: k in KNN, max_depth in trees, C & gamma in SVM, learning rate in boosting.
